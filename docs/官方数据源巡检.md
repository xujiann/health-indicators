# 官方数据源巡检

## 目标

对卫生健康、医疗保障和人口健康公开数据的官方发布页面进行定期可达性与内容指纹巡检。巡检用于发现新增公报、快报或栏目更新，不直接把网页内容写入公开指标库。

## 已纳入巡检的来源

| 机构 | 巡检内容 | 入库状态 |
| --- | --- | --- |
| 国家卫生健康委 | 卫生健康事业发展统计公报栏目 | 已覆盖 2015-2024 年；持续等待 2025 年公报 |
| 国家统计局 | 国民经济和社会发展统计公报 | 已覆盖 2025 年可稳定核验的卫生资源与服务量 |
| 国家医保局 | 医疗保障事业发展统计快报 | 已覆盖 2025 年参保、基金与待遇专题指标 |
| 国家疾控局 | 传染病疫情、疾控统计公开栏目 | 作为新增公共卫生指标候选来源，待按月度和年度口径复核 |

## 自动与人工边界

`scripts/check-official-sources.mjs` 会抓取登记的官方页面，输出页面标题、内容指纹、HTTP 状态和采集时间到 `tmp/official-source-watch.json`。GitHub Actions 每周运行一次并上传该结果；网络错误或服务端异常会以非零状态结束。对 401、403、412、429 等站点访问策略，结果标记为 `restricted`，提示改用人工浏览器复核，避免把反爬策略误判为数据源失效。

数据不会由脚本自动发布。新增条目必须在 `data/*-additions.json` 中完成以下复核后，才运行 `node scripts/update-public-data.mjs`：

1. 核验发布机构、发布日期和原始链接。
2. 确认统计年度、单位、指标定义及与既有 `compare_key` 的可比性。
3. 为快报、初步数、修订数或不同部门口径保留清晰的来源说明，不能覆盖非同口径记录。
4. 运行 `node scripts/validate-public-data.mjs`，确认无重复键、异常年份、异常数值或 PDF 断行碎片。

该安排使更新发现自动化，而公开数据的口径判断仍保持可追溯和可审计。
